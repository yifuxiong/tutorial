## 推理并行策略

推理场景的 7 种并行策略：TP、PP、DP、EP、DP Attention、SP、CP。

按切分维度分为四组：

Weight（参数维度）：TP、EP
Batch（样本维度）：DP、DP Attention
Sequence / Context（序列维度）：SP、CP
Layer（深度维度）：PP

### 1.为什么需要并行

单卡放不下或算不动模型时，把“某个东西”切到多卡。

推理场景选策略围绕三个目标：

- 放得下（模型 + KV cache 进显存）

- 跑得快（prefill 吞吐 / decode 延迟）

- 通信不成瓶颈（尤其跨机）

| # | 策略 | 切什么	| 解决什么问题 |
| - | --- | ------ | ----------- |
| 1	| TP | Weight 的 hidden 维 | 单层算不动 / 放不下 |
| 2	| PP | Layer（深度） | 整模型放不下（跨机） |
| 3	| DP | Batch（请求复制）| 扩吞吐（外层并行） |
| 4	| EP | MoE Expert | 超大 MoE 模型（专家切卡） |
| 5	| DP | Attention | Batch + attention heads 的小组切 | MoE 跨机时解耦 attention 和 MoE |
| 6	| SP | Sequence（TP 区外激活） | 省 TP 外的激活冗余显存 |
| 7	| CP | Sequence（attention 内部） | 超长 context（32K+） |

TP 和 EP 切参数，DP / DP Attention 切批次，SP / CP 切序列，PP 切深度。四个维度、互相正交、可以叠加。

### n.策略对比速查

| 策略 | 切什么 | 通信模式 | 通信频率 | 带宽需求 | 跨机友好 | 主要目的 |
| --- | ------ | -------- | ------- | ------- | ------- | -------- |
| TP | Hidden dim | AllReduce 激活 | 每层 2 次 | 极高 | ✗ | 单卡放不下/算不动 |
| PP | Layer | P2P Send/Recv | 每 stage 边界 | 低 | ✓ | 超大模型跨机部署 |
| DP | Batch（请求） | 无（前向独立） | — | — | ✓ | 扩展吞吐 |
| EP | Expert | A2A （dispatch/combine） | 每 MoE 层 2 次 | 高 | 一般 | MoE 模型扩展 |
| DP Attention | Head（沿 DP 维度） | RS + AG（attn_tp 组内） | 每 attention 层 RS+AG | 中 | ✓ | MoE 推理省 world AllReduce | 
| SP | Seq（非 attention 部分） | AllGather + ReduceScatter | 每层（与 TP 配合） | 等于 TP | ✗ | 省 prefill 激活显存 |
| CP | Seq（attention 内部） | Ring / AllGather K，V | 每 attention 层 | 	中-高 | ✓ | 超长序列 |
